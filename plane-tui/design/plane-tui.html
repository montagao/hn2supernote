Design notes — plane-tui

press ! to dismiss · numbered pins on the terminal
  1. The board tells the truth. Real counts from the API: 88 of 120 TMOM items sit in Backlog, 86 have no priority. The whole design optimizes for digging out of that.
  2. Projects are tabs, one keystroke away. 1/2/3 switches TMOM · TMIOS · MKT. No workspace-picker ceremony; the workspace is in the prompt.
  3. Priority is a one-cell glyph (‼ ↑ − ↓ ·) in a fixed column — scannable down a whole list without reading a word.
  4. Cursor ≠ marks. Reverse-video indigo is where you are; the ✓ column and left bar are what you've marked. Batch bar appears only when marks exist.
  5. Triage mode (T) is the killer feature. One bordered box, one item at a time, one keystroke per decision — priority or state, auto-advance, live progress meter. 88 items becomes ten minutes.
  6. Every mutation is a visible API call. Cards update optimistically; the header job segment spins (⣾) until the PATCH confirms with latency. Nothing mutates silently.
  7. The API drawer (x) teaches the REST API — method, path, payload, status, ms — the exact calls the plane-api skill makes. The TUI is a front-end for the API, and it shows its work.
  8. Inspector = everything without leaving home row. State, priority, labels, dates as relative + absolute, description, session activity, and the canonical /browse/TMOM-n URL (never the legacy /issues/ route).
  9. Single-key menus on the command line for state/priority/label — no popup dialogs; the prompt row becomes the menu, vim-style.
  10. The command line teaches keys. Every :command in the wildmenu shows its direct key, so the palette makes itself obsolete.
  11. Honest pagination. Columns show "… n more on server" instead of pretending the page is the world; R fetches the next page.
  12. Still a terminal, deliberately. One font, one size; bold/dim/color/reverse-video; square corners; braille spinners and ▰▱ meters. Every element maps to curses calls.
